# Orchestration Method

## Why This Project Uses Specs And Sprint Docs

This museum is small enough to look simple, but it is still vulnerable to drift. Without durable artifacts, design decisions live only in chat history and get lost across sessions.

Specs and sprint docs are used here to preserve intent.

## What A Spec Does Here

A spec defines the feature-level contract.

In this project, a spec explains:

- the curatorial problem being solved
- the design goals and visitor outcomes
- the architecture and artifact systems required
- the risks around scope, clutter, or historical flattening
- the verification method

## What A Sprint Doc Does Here

A sprint doc turns the larger contract into one bounded execution unit.

In this project, a sprint doc defines:

- one narrow goal
- the exact files and surfaces involved
- the tasks for that unit only
- the verification required before the sprint is considered done

## Core Workflow

1. define the feature with a spec
2. split it into small sprints
3. implement one sprint at a time
4. verify the result
5. record findings and deviations

## Why This Matters For A Curated Site

Strong museum sites do not emerge from random page edits. They emerge from controlled sequence, evidence discipline, and repeatable quality checks.
